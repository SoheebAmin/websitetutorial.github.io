Client-Side Javascript

Here are a few ways to use proragmming logic to offer more on your site

Run Programs

We can use Javascript to run functions with users providing input.

This is similar to what we can do with Flask, but nothing will go back to our servers since the code is run locally. We can leverage this programming power by having Javascript scramble the text in the input box. Let's also provide an error message if the box is left blank.


For an empty entry, we can switch between the alert error and preventing submission.

Control the Webpage

Using JavaScript, we can update the code that displays this page.

We can actually have JavaScript modify the wepage based on whatever requirements we want. For example, we can udpate the background or font color, font size, or font style. These are all done by creating style tags and injecting the desired CSS at the end of the head tag. We also force the style globally with the "!important" property. For overrides, it is better to learn CSS specificity, but this property is fine for temporary page changes like these ones.


A helper function checks if you color exists, then document.createElement injects the input.


This calls the same Javascript function as font color, with a different parameter (the input ID).


The input tag marks this as a type of number with a min and max to limit the range


document.CreateElement injects the selection from the drop down. Case insensitive.

Have the page update itself

We can use Javascript to update what our page shows, with or without events

Web pages that try to look alive use Javascript to have elements on the page update after a certain amount of time, such as a slideshow that will change pictures every few seconds. Pages can use event listeners to have elements update based on user actions, such as typing, clicking, scrolling, or even just moving the mouse.

All changes made on the page will be undone with a refresh (F5).
